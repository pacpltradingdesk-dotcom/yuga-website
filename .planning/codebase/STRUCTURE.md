## File Naming Conventions

- **Pages**: `page.tsx` — lowercase, matching Next.js App Router convention
- **Layout**: `layout.tsx` — single root layout
- **Components**: `PascalCase.tsx` — e.g. `Navbar.tsx`, `ServiceCard.tsx`, `HeroVideo.tsx`
- **Library/data**: `kebab-case.ts` — e.g. `company-data.ts`
- **Config files**: `kebab-case` with appropriate extension — e.g. `next.config.ts`, `postcss.config.mjs`, `eslint.config.mjs`
- **Test files**: Placed in `__tests__/` directory (Jest convention)
- **Route segments**: `kebab-case` directory names — e.g. `why-us/`, `pyrolysis/`

## Routing

All routing is file-system based via Next.js App Router. The site is exported as static HTML with `trailingSlash: true`.

| URL | File | Notes |
|---|---|---|
| `/` | `app/page.tsx` | Home — hero video, stats, service teasers |
| `/about` | `app/about/page.tsx` | Company overview, founder timeline |
| `/services` | `app/services/page.tsx` | Three service verticals with anchor nav |
| `/services#bio-bitumen` | `app/services/page.tsx` | Anchor scroll — Bio-Bitumen section |
| `/services#pmc` | `app/services/page.tsx` | Anchor scroll — Project Management section |
| `/services#it-solutions` | `app/services/page.tsx` | Anchor scroll — IT Solutions section |
| `/pyrolysis` | `app/pyrolysis/page.tsx` | Pyrolysis technology deep-dive |
| `/why-us` | `app/why-us/page.tsx` | Competitive differentiation page |
| `/contact` | `app/contact/page.tsx` | Contact info and form |

No dynamic routes (`[slug]` etc.) exist. No API routes. No middleware.

## Component Organization

All components live flat in `components/` — no subdirectories or grouping by feature/domain. Components fall into three functional categories:

**Layout components** (used in every page via `app/layout.tsx`):
- `Navbar.tsx` — sticky top bar with logo, nav links, mobile hamburger menu
- `Footer.tsx` — three-column footer: brand info, quick links, contact details

**Page-specific components** (used in a single page):
- `HeroVideo.tsx` — home page only; plays `public/hero.mp4` as full-viewport background
- `StatsBar.tsx` — home page only; animated counters for key company stats
- `TimelineSection.tsx` — about page only; renders career milestone cards from `CAREER_TRACK`
- `NetworkGrid.tsx` — why-us page only; renders `INDUSTRY_NETWORK` breakdown
- `ServiceCard.tsx` — services page only; renders a single bio-bitumen production stage
- `ContactForm.tsx` — contact page only; Formspree-powered form

**Shared/utility components** (designed for reuse):
- `TimelineItem.tsx` — renders a single `CareerEntry`; imported by `TimelineSection` (though `TimelineSection` renders its own card variant inline and `TimelineItem` may be unused in practice)
- `HeroSection.tsx` — legacy component, not currently used by any page

**Client vs Server split:**
- `"use client"`: `Navbar`, `HeroVideo`, `StatsBar`, `ContactForm` (need browser APIs or React state)
- Server (default): all other components

## Assets

**Video:**
- `public/hero.mp4` — the only local media file; used as the autoplay, muted, looping background for the home page hero via `<video src="/hero.mp4" />`

**Images:**
- No local image files. All images are loaded from Unsplash CDN (`https://images.unsplash.com/photo-*`) embedded as URLs in component JSX and in `lib/company-data.ts` (feedstock entries include `imgSrc` fields)
- `next/image` component is used throughout with `fill` layout and `images: { unoptimized: true }` in `next.config.ts`

**Fonts:**
- Inter (Google Fonts) loaded via `next/font/google` in `app/layout.tsx`; serif font used for the YUGA wordmark in `Navbar.tsx` via Tailwind `font-serif` class

**CSS:**
- `app/globals.css` — Tailwind CSS v4 base import
- All styling done via Tailwind utility classes inline in JSX (no CSS Modules, no separate stylesheet files per component)

**Environment variables:**
- `NEXT_PUBLIC_FORMSPREE_ID` — the Formspree form ID for contact form submissions (falls back to string `"YOUR_FORM_ID"` if not set)
